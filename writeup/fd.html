What is a file descriptor in Linux?

A file descriptor is an integer handle that a process uses to access files and sockets or other I/O resources. FD isn't the file itself, but a number the OS uses to keep track of which open file or resource the process is dealing with

By default, each process starts with three standard FDs:

  • 0 - stdin (input) – what the user types from keyboard
  • 1 - stdout (output) – output on the screen
  • 2 - stderr (error messages) – where error messages go

When you call open() (a system call), the kernel returns the next available FD or -1 in case of an error. We can also use read() to get data or write() to send data.

Example C definition of read:

ssize_t read(int fd, const void *buf, size_t count);
  • fd : the file descriptor to read from (we can put 0 for stdin to read from user's input).
  • buf: the buffer to where the data will go.
  • count: number of bytes to read. Note: don't put a larger number then what the buffer can hold to avoid buffer overflow.

This allows interaction with files/resources without worrying about memory addresses.

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

char buf[32];
int main(int argc, char* argv[], char* envp[]){
    if(argc < 2){
        printf("pass argv[1] a number\n");
        return 0;
    }
    int fd = atoi(argv[1]) - 0x1234;
    int len = 0;
    len = read(fd, buf, 32);
    if(!strcmp("LETMEWIN\n", buf)){
        printf("good job :)\n");
        setregid(getegid(), getegid());
        system("/bin/cat flag");
        exit(0);
    }
    printf("learn about Linux file IO\n");
    return 0;
}

Explanation:

  • The program takes a parameter as FD, subtracts 0x1234 (4660 in decimal).
  • It reads 32 bytes from that FD into buf.
  • If buf matches "LETMEWIN\n", it prints the flag.

We need fd=0 for stdin. Since input is subtracted by 4660, then we will input it to the program, and then we can enter the string "LETMEWIN":

./fd 4660
LETMEWIN
